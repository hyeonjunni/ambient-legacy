## 2. EXAONE Findings Relevant to This Project

EXAONE 3.5 is a bilingual Korean-English instruction-tuned model family. The official README describes 2.4B, 7.8B, and 32B variants, all with long-context support up to 32K tokens.

The 2.4B config downloaded locally shows:

- architecture: `ExaoneForCausalLM`
- layers: `30`
- hidden size: `2560`
- attention heads: `32`
- key/value heads: `8`
- context length: `32768`
- vocab size: `102400`
- tied embeddings: `true`

The Transformers path uses `trust_remote_code=True` because `config.json` maps:

- `AutoConfig` -> `configuration_exaone.ExaoneConfig`
- `AutoModelForCausalLM` -> `modeling_exaone.ExaoneForCausalLM`

The chat template uses EXAONE-specific turn markers:

```text
[|system|]... [|endofturn|]
[|user|]...
[|assistant|]...
```

This matters because our SFT data should remain in chat `messages` format and let the tokenizer apply the model-specific chat template. Manually flattening all rows into another prompt syntax would make EXAONE comparison less reliable.

## 3. License Caution

EXAONE 3.5 uses `EXAONE AI Model License Agreement 1.1 - NC`.

Important implications for this project:

- The model is allowed for research use.
- Commercial use is prohibited without a separate license.
- The license also restricts using the model, derivatives, or outputs to develop or improve other models.
- Because of this, I did not use EXAONE-generated outputs to create the synthetic dataset.

The expanded dataset below was generated by deterministic local templates, not by EXAONE model output.

## 4. Similar Public Dataset Search

Potentially relevant public references:

### KoAlpaca-RealQA

- URL: https://huggingface.co/datasets/beomi/KoAlpaca-RealQA
- Korean instruction dataset based on real Korean user interactions with ChatKoAlpaca.
- Responses are generated with GPT-4o.
- License: `CC-BY-SA-4.0`.
- Hugging Face page notes gated access conditions.
- Useful as a style reference for natural Korean instruction phrasing, but not a direct fit for our grounded family-memory RAG task.

### KoQuality

- URL: https://huggingface.co/datasets/DILAB-HYU/KoQuality
- Korean instruction dataset selected from several instruction datasets.
- The dataset card states 4,006 final instructions selected from 384,067 integrated rows.
- Useful as a quality/filtering reference: do not just maximize row count; cluster and sample for diversity.

### Korean Instruction Dataset Collection

- URL: https://huggingface.co/collections/beomi/korean-instruction-dataset
- Collection of Korean instruction datasets.
- Useful for discovering general Korean instruction formats.
- Still needs license-by-license review before importing any rows.

### KIT-19

- URL: https://aclanthology.org/2024.lrec-main.853.pdf
- Korean instruction toolkit across 19 tasks.
- The paper is useful for task diversification and instruction-template design.
- It is broader than our use case, but its template-diversification method is relevant.

### ChatQA / Conversational RAG

- URL: https://proceedings.neurips.cc/paper_files/paper/2024/file/1c0d54ebd0a6e58c4eca7d591e374b9d-Paper-Conference.pdf
- Useful because it explicitly studies conversational QA, retrieval context, and unanswerable cases.
- The key direction for us is to include a meaningful share of "record does not confirm this" rows to reduce hallucination.

## 5. Expanded Synthetic Dataset

Created a new deterministic generator:

- `backend/scripts/build_weekly_sft_augmented.py`

Generated dataset:

- `backend/finetune/weekly_sft_augmented_2026-05-30/train.jsonl`
- `backend/finetune/weekly_sft_augmented_2026-05-30/valid.jsonl`
- `backend/finetune/weekly_sft_augmented_2026-05-30/test.jsonl`
- `backend/finetune/weekly_sft_augmented_2026-05-30/manifest.json`

The dataset uses the same `messages` JSONL format expected by `mlx-lm`.

Counts:

```json
{
  "source_cases": 37,
  "total_rows": 592,
  "split_counts": {
    "train": 473,
    "valid": 59,
    "test": 60
  },
  "category_counts": {
    "recall_summary": 128,
    "unanswerable": 80,
    "metadata_guardrail": 80,
    "missing_detail_guardrail": 128,
    "structured_summary": 80,
    "conflict_resolution": 96
  }
}
```

The generator varies:

- 37 base family-memory cases
- 4 persona markdown profiles
- 4 query phrasings per case
- 6 behavior categories

Behavior categories:

- `recall_summary`: summarize retrieved family memories
- `missing_detail_guardrail`: refuse unsupported exact dates, names, addresses, prices
- `conflict_resolution`: present conflicting records without deciding
- `structured_summary`: turn recipes and notes into ordered steps
- `metadata_guardrail`: avoid exposing tags, confidence, source labels, persona markdown, internal reasoning
- `unanswerable`: say the current record does not confirm the requested fact

## 6. Why This Shape Fits Ambient Legacy

Generic Korean instruction datasets can help with natural phrasing, but they do not directly teach the core behavior this app needs:

- answer from retrieved family records
- avoid turning `saved_at` into event date
- refuse unsupported detail
- handle OCR/text conflicts
- hide internal retrieval metadata
- keep family-chat tone

For that reason, the first expanded dataset is domain-shaped synthetic data rather than imported general instruction rows.
