# Thursday Demo Scope

## 결론

목요일 데모 전까지는 아래 4가지가 끊기지 않게 동작하면 충분하다.

1. 로그인 또는 데모 로그인
2. 데모 가족방과 샘플 기록 자동 준비
3. 모델 선택 + 페르소나 선택
4. 질문 전송 시 근거와 함께 응답 반환

즉, `완전한 온디바이스 추론`이나 `실제 OCR 고정밀 처리`보다  
`유산 서비스의 구조와 방향이 명확히 보이는 시연 흐름`을 먼저 확보하는 것이 맞다.

## Must Have

### 1. 데모 로그인

- Google 로그인 또는 데모 로그인
- 사용자 프로필 동기화

### 2. 데모 데이터 준비

- 버튼 한 번으로 가족방 생성
- 샘플 텍스트/이미지/영상 기록 자동 생성
- 저장소와 AI 데모에서 바로 활용 가능

### 3. AI 설정 데모

- 모델 선택
- 페르소나 선택
- 현재 응답이 온디바이스 계층인지, 가족 금고 정본 계층인지 표시

### 4. AI 질문 응답

- 앱 -> backend `/ai/chat-demo`
- backend -> prompt package 조립
- provider adapter 선택
- 응답 + evidence 반환

### 5. 발표 스토리

- "기기가 바뀌어도 유산은 남는다"
- "모델은 바뀌어도 페르소나는 남는다"
- "가족 공용 정본과 개인 맞춤형 모델을 분리했다"

## Nice to Have

- 실제 Gemma / EXAONE 엔드포인트 연결
- OCR 결과를 실업로드 데이터에서 자동 반영
- 응답 출처를 시각적으로 더 풍부하게 표시
- 추천 질문 버튼
