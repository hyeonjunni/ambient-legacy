# LLM Weekly Goal Plan

## Goal

This week should prove three things with small, repeatable checks:

1. Backend DB context is reaching the LLM prompt.
2. Prompt engineering reduces unsupported details and prompt leakage.
3. Gemma remains the default candidate after a direct Gemma vs EXAONE comparison.

## Recommendation

Use `Gemma 4 E2B` as the default model and `EXAONE 3.5 7.8B` as the fast comparison model.

The existing research notes already show:

- `Gemma 4 E2B`: most stable default, no prompt leakage in top-3 benchmark.
- `EXAONE 3.5 7.8B`: faster, but occasional prompt discipline issues.
- `Qwen 3 8B`: useful quality reference, but too slow for the default demo path.

## Fine-Tuning Scope

Do not start with full LoRA/SFT weight training this week unless there is spare GPU time.

Start with:

- prompt-tuned Ollama adapter models from `backend/llm_adapters/*.Modelfile`
- a small SFT seed JSONL generated by `backend/scripts/llm_weekly_experiment.py`
- the same DB-grounded and fixture-grounded cases for base vs tuned comparison

This gives quick signal without risking the app/server path.

## Success Criteria

- `/api/v1/system/health/db` returns connected on the running backend.
- Evaluation output shows DB rows selected as retrieved evidence.
- Gemma tuned adapter has no prompt leakage and does not invent unsupported details.
- EXAONE tuned adapter reduces prompt leakage compared with the base model.
- The report records both latency and answer quality flags.

## Commands

Start Ollama with the repo-local model store:

```bash
OLLAMA_MODELS=/Users/hyeonjun/ambient_legacy/.ollama-models ollama serve
```

Create the adapter models:

```bash
OLLAMA_MODELS=/Users/hyeonjun/ambient_legacy/.ollama-models ollama create ambient-legacy-gemma-e2b-weekly -f backend/llm_adapters/gemma_e2b_weekly.Modelfile
OLLAMA_MODELS=/Users/hyeonjun/ambient_legacy/.ollama-models ollama create ambient-legacy-exaone-35-78b-weekly -f backend/llm_adapters/exaone_35_78b_weekly.Modelfile
```

Run the comparison:

```bash
python3 backend/scripts/llm_weekly_experiment.py --write-sft
```
